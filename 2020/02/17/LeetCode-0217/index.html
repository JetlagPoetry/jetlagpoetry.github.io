121. Best Time to Buy and Sell Stock

Easy

Problem Description:

Say you have an array for which the i-th element is the price of a given stock on day i.
If you were only permitted to complete at most one transaction (i.e., buy one and sell one share of the stock), design an algorithm to find the maximum profit.
Note that you cannot sell a stock before you buy one.

My Solution:

Memorize 3 variable:

  1. max profit we can make up till element i
  2. min price we see ever
  3. max price we see since we see min

Iterate over each element:

  • If the price is smaller than min, update maxProfit with larger value of maxProfit and max-min, update min and max to be the current value.
  • If the price is larger than max, update max.

Do not forget to collect the last max-min, and return maxProfit.

122. Best Time to Buy and Sell Stock II

Easy

Problem Description:

Say you have an array for which the ith element is the price of a given stock on day i.
Design an algorithm to find the maximum profit. You may complete as many transactions as you like (i.e., buy one and sell one share of the stock multiple times).
Note: You may not engage in multiple transactions at the same time (i.e., you must sell the stock before you buy again).

My Solution:

Memorize 3 variable:

  1. total profit we make till now
  2. min price we see ever
  3. max price we see since we see min

Iterate over each element:

  • If the price is smaller than min, add max-min to sum, update min and max to be the current value.

Do not forget to collect the last max-min, and return sum.

125. Valid Palindrome

Easy

Problem Description:

Given a string, determine if it is a palindrome, considering only alphanumeric characters and ignoring cases.

Note: For the purpose of this problem, we define empty string as valid palindrome.

My Solution:

  1. Delete all characters other than digits and letters.
  2. Convert the string to lowercase.
  3. Examine if each char i is the same with char len-i-1. Can also do this by reversing string and check if 2 strings are the same.

155. Min Stack

Easy

Problem Description:

Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

  • push(x) – Push element x onto stack.
  • pop() – Removes the element on top of the stack.
  • top() – Get the top element.
  • getMin() – Retrieve the minimum element in the stack.

My Solution:

For each node, store current value, min value of stack after pushing this node, and link to the next node.

When pushing a node, determine the min value to be minimum value of the current stack head and current node value. Set each attribute and update the stack top.
