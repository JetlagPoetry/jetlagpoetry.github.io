39. Combination Sum

Posted on 2020-06-09 Edited on 2020-06-11 In LeetCode , 1-100题
backtrack,DP

Medium

题目描述:

Given a set of candidate numbers (candidates) (without duplicates) and a target number (target), find all unique combinations in candidates where the candidate numbers sums to target.

The same repeated number may be chosen from candidates unlimited number of times.

Note:

All numbers (including target) will be positive integers.
The solution set must not contain duplicate combinations.

思路一:

  • 回溯法
  • 使用递归不断向下创建combination，将一个新的candidate number添加到本节点已有的list中，直到超过target时停止，等于target时添加到答案集合中。 需要注意的是，每个节点内可添加的数字并不是全部candidate，我们需要注意candidate的顺序，否则会出现重复项。
  • 例 candidates = [2,3,6,7], target = 7,
    1. 当前路径是[2]时，那么接下来分别添加2、3、6、7四个元素，得到[2,2],[2,3],[2,6],[2,7]四个节点，后两个组合大于target被丢弃。
    2. 当前路径是[2,3]时，我们规定不能添加比最后一个元素更小的数了，那么分别添加3、6、7后，得到[2,3,3],[2,3,6],[2,3,7]，同样后两个大于target。

思路二：

  • 动态规划
  • 对于target=n的情况来说，依此构建并存储target=0至target=n-1时的combination。
  • 初始时，只有opt(0)=[[]]。在每一步，将candidate number与可能的每个数字相加，并在他们之和的元素里添加可能组成的所有列表。
  • 例如当前candidate是3，target=7，那么需要考虑的情况有0,1,2,3,4。
    0–>在opt(3)中添加[3]
    1–>在opt(4)中添加[opt(1)[0],3],[opt(1)[1],3],[opt(1)[n],3]
    2–>在opt(5)中添加[opt(2)[n],3]
    3–>在opt(6)中添加[opt(3)[n],3]
    4–>在opt(7)中添加[opt(4)[n],3]
  • 全部candidate遍历过后（只遍历一次，需要保证candidate有顺序），得到opt(target)返回。
