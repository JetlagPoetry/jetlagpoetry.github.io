6. ZigZag Conversion

Posted on 2020-06-04 Edited on 2020-06-05 In LeetCode , 1-100题

Medium

题目描述:

The string “PAYPALISHIRING” is written in a zigzag pattern on a given number of rows like this: (you may want to display this pattern in a fixed font for better legibility)

And then read line by line: “PAHNAPLSIIGYIR”

思路:

  • 画风清奇的一道题，可以当做找规律的题来做。
  • 对于总共n行的zigzag，不难发现每(2n-2)个元素为一个周期。
  • 假设当前为第j个周期，坐落在第i行的元素为：
    1. 当i为第0行或第n-1行时，只有一个字符，分别是j*(2n-2)和j*(2n-2)+n-1。
    2. 当i是中间行时，每个周期包含两个字符，分别是j*(2n-2)+i和(j+1)*(2n-2)-i。
  • 因此从第0行到第n-1行，通过循环依此添加该行所有的字符即可。
  • 需要注意的是，如果n=1，那么很难跳出循环，所以需要在函数首部进行额外的检查。

Medium

题目描述:

Given a binary tree, return the level order traversal of its nodes’ values. (ie, from left to right, level by level).

For example:
Given binary tree [3,9,20,null,null,15,7],

return its level order traversal as:

思路:

  • 比较基础，使用队列进行二叉树的层级遍历。
  • 每次循环时，队列中存储树的某一层的全部元素。依次将元素弹出，添加元素至答案列表，并将它的左右子节点加入至新队列。

###API

  • 队列初始化：Queue last = new LinkedList<>();
  • 嵌套列表初始化:List<List> list = new ArrayList<List>();
  • 队列的操作：
    Throws exception Returns special value
    Insert add(e) offer(e)
    Remove remove() poll()
    Examine element() peek()
  • 另外，支持从Collection中继承的函数：addAll, clear, contains, containsAll, isEmpty, equals等。

99. Recover Binary Search Tree

Posted on 2020-06-03 Edited on 2020-06-04 In LeetCode , 1-100题

High

题目描述:

Two elements of a binary search tree (BST) are swapped by mistake.
Recover the tree without changing its structure.

思路:

  • 对于一个二叉搜索树，中序遍历得到的结果应当是从小至大的元素值排列，例如[1,2,3,4,5,6]。
  • 当有两个元素的值被调换时，只有两种情况。
    1. 两个元素相邻：在从头至尾两两配对比较时，会有一对的大小关系错误，例如[1,3,2,4,5,6]。
    2. 两个元素不相邻：在两两比较时，会有两对大小关系错误，例如[1,5,3,4,2,6]中的[5,3]和[4,2]。
  • 那么本题的递归解法是，对BST进行中序遍历，遍历时有一个对象记住上一个被遍历的节点。对于第一种情况，直接记录两个节点的值；对于第二种情况，分别记录第一对的第一个元素，和第二对的第二个元素。在遍历后对两个节点的值进行调换。

98. Validate Binary Search Tree

Posted on 2020-06-03 In LeetCode , 1-100题

Medium

题目描述:

Given a binary tree, determine if it is a valid binary search tree (BST).

Assume a BST is defined as follows:

  • The left subtree of a node contains only nodes with keys less than the node’s key.
  • The right subtree of a node contains only nodes with keys greater than the node’s key.
  • Both the left and right subtrees must also be binary search trees.

思路:

  • 先序遍历实现
  • 创建新函数，对于每个节点的Valid检查，需要传入当前节点预期拥有的最大值与最小值。首先检查当前节点，是否超出区间，如果超出返回false。之后分别对左子节点和右子节点进行检查。对于左子树来说，新的取值范围为(min, root.val)；类似地，对于右子树来说取值范围是(root.val, max)。通过对左右子树的BST检查，当两者都返回true时，向上级返回true。
  • 对于根节点如何设置取值范围呢？参考了其他人的答案，使用了Long.MAX_VALUE与Long.MIN_VALUE，并且所有界限均使用long型。
