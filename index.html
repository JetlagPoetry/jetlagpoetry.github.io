102. Binary Tree Level Order Traversal

Posted on 2020-06-03 Edited on 2020-06-04 In LeetCode , 101-200题

Medium

题目描述:

Given a binary tree, return the level order traversal of its nodes’ values. (ie, from left to right, level by level).

For example:
Given binary tree [3,9,20,null,null,15,7],

return its level order traversal as:

思路:

  • 比较基础，使用队列进行二叉树的层级遍历。
  • 每次循环时，队列中存储树的某一层的全部元素。依次将元素弹出，添加元素至答案列表，并将它的左右子节点加入至新队列。

###API

  • 队列初始化：Queue last = new LinkedList<>();
  • 嵌套列表初始化:List<List> list = new ArrayList<List>();
  • 队列的操作：
    Throws exception Returns special value
    Insert add(e) offer(e)
    Remove remove() poll()
    Examine element() peek()
  • 另外，支持从Collection中继承的函数：addAll, clear, contains, containsAll, isEmpty, equals等。

99. Recover Binary Search Tree

Posted on 2020-06-03 Edited on 2020-06-04 In LeetCode , 1-100题

High

题目描述:

Two elements of a binary search tree (BST) are swapped by mistake.
Recover the tree without changing its structure.

思路:

  • 对于一个二叉搜索树，中序遍历得到的结果应当是从小至大的元素值排列，例如[1,2,3,4,5,6]。
  • 当有两个元素的值被调换时，只有两种情况。
    1. 两个元素相邻：在从头至尾两两配对比较时，会有一对的大小关系错误，例如[1,3,2,4,5,6]。
    2. 两个元素不相邻：在两两比较时，会有两对大小关系错误，例如[1,5,3,4,2,6]中的[5,3]和[4,2]。
  • 那么本题的递归解法是，对BST进行中序遍历，遍历时有一个对象记住上一个被遍历的节点。对于第一种情况，直接记录两个节点的值；对于第二种情况，分别记录第一对的第一个元素，和第二对的第二个元素。在遍历后对两个节点的值进行调换。

98. Validate Binary Search Tree

Posted on 2020-06-03 In LeetCode , 1-100题

Medium

题目描述:

Given a binary tree, determine if it is a valid binary search tree (BST).

Assume a BST is defined as follows:

  • The left subtree of a node contains only nodes with keys less than the node’s key.
  • The right subtree of a node contains only nodes with keys greater than the node’s key.
  • Both the left and right subtrees must also be binary search trees.

思路:

  • 先序遍历实现
  • 创建新函数，对于每个节点的Valid检查，需要传入当前节点预期拥有的最大值与最小值。首先检查当前节点，是否超出区间，如果超出返回false。之后分别对左子节点和右子节点进行检查。对于左子树来说，新的取值范围为(min, root.val)；类似地，对于右子树来说取值范围是(root.val, max)。通过对左右子树的BST检查，当两者都返回true时，向上级返回true。
  • 对于根节点如何设置取值范围呢？参考了其他人的答案，使用了Long.MAX_VALUE与Long.MIN_VALUE，并且所有界限均使用long型。

94. Binary Tree Inorder Traversal

Posted on 2020-06-01 Edited on 2020-06-03 In LeetCode , 1-100题

Medium

Tree Traversal

(a) Inorder (Left, Root, Right) - 中序
(b) Preorder (Root, Left, Right) - 前序
(c) Postorder (Left, Right, Root) - 后序

题目描述:

Given a binary tree, return the inorder traversal of its nodes’ values.

思路一:

  • 简单递归实现中序遍历
  • 在每个节点，将左节点返回结果、当前节点的值、右节点返回结果依此添加到当前集合中并返回。
  • 时间复杂度：平均情况下，O(h)=(logN)；最坏情况下，O(h)=O(n)。
  • 空间复杂度：O(h)

代码一

思路二:

  • 使用循环和栈来模拟递归过程
  • 初始化空栈，当前节点设置为根节点。 每次循环时做两件事：一，不断地将当前节点最左侧的一系列子节点推入战中，直到没有左子节点。 二，步骤一做完后，弹出栈中最顶元素（想象为当前子树的左侧最远孩子），处理这个最顶元素（本题中是将节点值加入list），之后将右子节点设置为当前节点。当节点全部遍历完且栈为空时停止循环。
  • 时间复杂度：平均情况下，O(h)=(logN)；最坏情况下，O(h)=O(n)。
  • 空间复杂度：O(h)

2. Add Two Numbers

Posted on 2020-05-27 Edited on 2020-06-03 In LeetCode , 1-100题

Medium

题目描述:

You are given two non-empty linked lists representing two non-negative integers. The digits are stored in reverse order and each of their nodes contain a single digit. Add the two numbers and return it as a linked list.

You may assume the two numbers do not contain any leading zero, except the number 0 itself.

思路:

  • 基础链表题目。
  • 使用两个指针分别指向l1和l2的节点，从头部开始。 循环相加每个pair，用布尔值记录进位信息。
  • 当其中一个链表被遍历完时，再循环加另一个非空的链表，同样记录每一步的进位信息。

3. Longest Substring Without Repeating Characters

Posted on 2020-05-27 Edited on 2020-06-03 In LeetCode , 1-100题

Medium

题目描述:

Given a string, find the length of the longest substring without repeating characters.

O(n) time: Use a string(hashset is better) to store current non-repeated substring. If current char is contained in the substring indexed at last, slide the left bound to last+1. Otherwise, increase the right bound and add current char to the string/set. Return the max value.

思路:

  • 使用一个字符串str来存储当前的无重复字符(推荐使用hashset)。循环遍历每个字符，如果当前字符在str中存在且位置为last，则重置字符串起始于last+1；否则，将当前字符添加到str末尾并记录当前字符串长度，返回最大值。
  • 时间复杂度：O(n)

46. Permutations

Medium

题目描述:

Given a collection of distinct integers, return all possible permutations.

思路:

  • 使用递归构建，函数传入整个nums数组与数字头部指针，每次运算n个元素的permutation是由后n-1个元素的permutation生成。
  • 例：已知由元素2和3生成的permutation有[2,3], [3,2]。 那么对于每个结果，将新的元素1插入到不同的位置。
    [2,3] -> [1,2,3], [2,1,3], [2,3,1]
    [3,2] -> [1,3,2], [3,1,2], [3,2,1]
